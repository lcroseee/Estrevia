# Онбординг и пользовательские флоу

---

## Принцип: Value Before Registration

> Пользователь получает ценность (расчёт карты) **до** регистрации. Регистрация = сохранение, а не доступ.

```
Визит (Landing / Direct)
  → Ввод даты рождения (без регистрации!)
    → Мгновенный расчёт натальной карты
      → «Сохранить карту?» → Регистрация
        → Сохранённая карта → Эссе → Лунный календарь
          → Premium предложение (через 3-5 сессий)
```

---

## Флоу 1: Новый посетитель (Landing Page)

```
1. Landing page
   ├── Hero: «Узнай свою настоящую карту звёзд»
   ├── CTA: Ввод даты рождения (3 поля: дата, время, город)
   └── Социальное доказательство (counter: «12,345 карт рассчитано»)

2. Расчёт (мгновенный, WASM на клиенте)
   ├── Анимация: колесо строится на экране
   ├── Результат: натальная карта (sidereal по умолчанию)
   ├── Toggle: «Показать тропическую для сравнения»
   └── Вопрос: «Ваш сидерический знак Солнца — Рыбы, а не Овен. Удивлены?»

3. Engagement
   ├── «Узнать больше про Солнце в Рыбах» → эссе (бесплатное)
   ├── «Сохранить карту» → регистрация (Clerk)
   ├── «Поделиться» → share card image (без регистрации)
   └── «Сравнить с другом» → (Фаза 2, требует регистрации)

4. Регистрация (Clerk)
   ├── Google OAuth (1 клик)
   ├── Email + magic link
   └── Email + пароль
```

### Критические решения

| Решение | Выбор | Почему |
|---------|-------|--------|
| Расчёт без регистрации | ✅ Да | Снижает friction. Конверсия visit→chart 40-60% |
| Sidereal по умолчанию | ✅ Да | Наш differentiator. Toggle для тропической |
| Время рождения обязательно | ❌ Нет | Опционально. Без времени = нет домов, но знаки есть |
| Показать «ваш знак другой» | ✅ Да | WOW-момент. Удержание внимания |

## Флоу 2: Зарегистрированный пользователь (возврат)

```
1. Открытие PWA / сайта
   ├── Если PWA installed → сразу в приложение
   └── Если браузер → auto-login (Clerk session)

2. Dashboard
   ├── Натальная карта (primary)
   ├── Лунная фаза сегодня
   ├── «Сегодня Луна в Скорпионе» (одна строка)
   └── Лента: последние события NASA/USGS

3. Навигация
   ├── [Карта] — натальное колесо
   ├── [Луна] — лунный календарь
   ├── [Эссе] — тексты по позициям
   ├── [Данные] — NASA/USGS лента
   └── [Ещё] — настройки, профиль, premium
```

## Флоу 3: Конверсия в Premium

```
Триггер: пользователь нажимает на locked функцию
  или: 5-я сессия (soft prompt)
  или: попытка синастрии (Фаза 2)

1. Premium modal
   ├── «Раскрой полную карту»
   ├── Что включено в Star ($9.99/мес)
   │   ├── Все 120 эссе (сейчас доступно 3: Солнце + Луна + Восходящий)
   │   ├── Транзиты и совместимость
   │   ├── Древо Жизни
   │   └── AI-аватар
   ├── Годовая подписка: $79.99/год (скидка 33%)
   └── CTA: «Начать» → Stripe Checkout

2. После оплаты
   ├── Confetti / celebration анимация
   ├── Все эссе разблокированы
   └── Welcome email (Resend)
```

### Free vs Premium границы (MVP)

| Функция | Free | Star |
|---------|------|------|
| Натальная карта | ✅ | ✅ |
| Эссе по позициям | 3 (Солнце + Луна + Восходящий) | Все 120 |
| Лунный календарь | ✅ | ✅ |
| Планетарные часы | ✅ | ✅ |
| NASA/USGS лента | ✅ | ✅ |
| Сохранение карт | 1 карта | Безлимитно |
| Sidereal/Tropical toggle | ✅ | ✅ |
| Экспорт карты (PNG) | ❌ | ✅ |

## Флоу 4: PWA Installation

```
1. Второй визит (или ручной prompt)
   ├── Браузер показывает install banner (Chrome/Edge)
   └── Кастомный prompt: «Установить Estrevia для быстрого доступа?»

2. После установки
   ├── Standalone window (без browser chrome)
   ├── Иконка на домашнем экране
   ├── Splash screen с логотипом
   └── Offline: расчёт карты работает без интернета
```

## Ошибки и Edge Cases

| Сценарий | Как обрабатываем |
|----------|-----------------|
| Время рождения неизвестно | Показываем карту без домов. Пояснение: «Время рождения влияет на дома и восходящий знак» |
| Город не найден | Автокомплит + ручной ввод координат |
| Дата в будущем | Блокируем. «Натальная карта рассчитывается для прошлых дат» |
| Дата до 1800 | Предупреждение: «Точность эфемерид снижается для дат до 1800 года» |
| Offline при расчёте | WASM работает offline. Показываем badge «Offline mode» |
| Offline при чтении эссе | Показываем кэшированное или «Текст доступен при подключении» |
