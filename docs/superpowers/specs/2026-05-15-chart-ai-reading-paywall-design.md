# Chart AI Reading + Paywall (Anon Funnel Fix)

**Date:** 2026-05-15
**Owner:** Kirill
**Status:** Design ‚Äî pending implementation plan
**Severity:** P0 (Meta-ad conversion leak)

## Problem

`/chart` is the top landing surface for cold Meta-ad traffic. Today the page renders the chart wheel, position table, Cosmic Passport, and Avatar generator unconditionally ‚Äî no paywall surface anywhere for anonymous or signed-in free users.

`ChartDisplay.tsx` was last touched at `876f1f2` (Apr 2026, pre-paywall-CRO commit batch). The recently-shipped value-then-block paywalls (Celtic Cross, three-card spread, Synastry AI) cover *secondary* routes that cold ad traffic typically never reaches.

Result: anon user from Meta ‚Üí calculates chart ‚Üí gets full free value (wheel + table + passport) ‚Üí leaves. Zero in-context upgrade ask.

## Scope

**In:** Add a new gated feature ‚Äî **AI Natal Chart Reading** ‚Äî to `/chart`, using the value-then-block pattern proven on Celtic Cross:

1. Free user sees a deterministic Sun/Moon/Asc teaser + blurred locked preview + `PaywallCta`.
2. Pro user sees the teaser + a "Generate Reading" button that calls a new premium endpoint returning a 6‚Äì8 paragraph LLM synthesis.
3. The reading is cached per `(chartId, locale)` in a new Postgres table so revisits are free.
4. Placement: between the wheel/table tabs and the Cosmic Passport section in `ChartDisplay`.

**Out:**
- Re-positioning Passport / Avatar (`PassportSection` and `AvatarSection` retain their current order).
- Prompt-tuning iteration loop ‚Äî first prod prompt ships as a skeleton; founder iterates after smoke test.
- A/B framework or feature flag ‚Äî direct-to-main per CLAUDE.md.
- New analytics events beyond `CHART_READING_GENERATED` and the existing paywall funnel dimensions.
- Multi-language reading beyond EN + ES.
- "Save reading to dashboard" UX ‚Äî caching by `chartId` is sufficient for revisit; future spec for saved readings list.
- Free preview generated by LLM ‚Äî explicitly rejected in design phase (cost + abuse risk).

## Decisions

| # | Decision | Rationale |
|---|---|---|
| D1 | **Mirror celtic-cross architecture faithfully** | 10-commit pattern already shipped, code-reviewer load minimised, bisectable revert |
| D2 | **Value-then-block ‚Äî deterministic teaser** | Zero LLM cost on free; strong visual cue via blurred locked preview |
| D3 | **Pro flow uses explicit "Generate" button**, not auto-fire on chart load | Cleaner engagement signal in PostHog; avoids LLM cost on every chart revisit even before cache hit; matches Passport's "Create Passport" click pattern |
| D4 | **Cache reading in Postgres** (`chart_readings` table) keyed by `(chart_id, locale)` | `natal_charts` already persists temp charts; cascade-delete via `ON DELETE CASCADE` reuses existing 90-day cleanup cron |
| D5 | **Placement between Wheel/Table and Passport** | Highest impression rate after the chart itself; Passport stays second (viral retention preserved) |
| D6 | **Pro reading scope: full synthesis (12 bodies + houses + top 3 aspects)** | $9.99/mo trial conversion needs deep enough perceived value; ~2500 token output on Claude Sonnet ‚âà $0.03 per cache-miss |
| D7 | **PaywallTrigger = `'natal-chart'`** | One extra union value reuses entire existing `PaywallCta` / `PaywallModal` flow |
| D8 | **Anon flow inherits existing Clerk sign-up bounce** | `PaywallModal` already detects 401/sign-out ‚Üí routes to `/sign-up?redirect_url=/checkout/start?...`. No new auth code |

### Server endpoint `/api/v1/chart/interpret`

**Mirrors `/api/v1/tarot/interpret/route.ts` (165 lines).** Differences:

| Step | tarot/interpret | chart/interpret |
|---|---|---|
| Body schema | `cards: array, spreadType` | `chartId: string, locale: enum` |
| Rate limit bucket | `tarot/interpret` (existing) | `chart/interpret` (new, same config: 10/min sliding window) |
| Cache check | none | **SELECT from `chart_readings` WHERE chart_id + locale** |
| Data source | request body | **SELECT chart_data FROM natal_charts WHERE id** |
| Prompt builder | `buildPrompt` (in route file) | `buildChartInterpretationPrompt` (in `astro-engine/lib/`) |
| Max tokens | 600 | **2500** |
| Cache write | none | **INSERT chart_readings ... ON CONFLICT DO NOTHING** (race-safe, non-fatal on failure) |
| Response data | `{ interpretation }` | `{ reading, source: 'cache' \| 'generated' }` |

Auth, Anthropic call, error mapping, Sentry capture ‚Äî identical to tarot endpoint.

**Error response codes (mirror tarot for consistency):**
- `401 UNAUTHORIZED` ‚Äî no Clerk session
- `403 FORBIDDEN` ‚Äî session but no Pro (returned by `requirePremium()`)
- `400 VALIDATION_ERROR` / `INVALID_JSON` ‚Äî bad body
- `404 CHART_NOT_FOUND` ‚Äî chartId not in `natal_charts`
- `429 RATE_LIMITED`
- `502 AI_SERVICE_ERROR` / `EMPTY_RESPONSE` ‚Äî Anthropic call failed
- `503 SERVICE_UNAVAILABLE` ‚Äî `ANTHROPIC_API_KEY` env missing
- `500 INTERNAL_ERROR` ‚Äî Sentry-reported

### `chart_readings` table

**Migration `drizzle/0010_chart_readings.sql`:**
```sql
CREATE TABLE chart_readings (
  id text PRIMARY KEY,
  chart_id text NOT NULL REFERENCES natal_charts(id) ON DELETE CASCADE,
  locale text NOT NULL CHECK (locale IN ('en','es')),
  body text NOT NULL,
  model text NOT NULL DEFAULT 'claude-sonnet-4-20250514',
  generated_at timestamptz NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX chart_readings_chart_locale_uniq ON chart_readings(chart_id, locale);
CREATE INDEX chart_readings_chart_id_idx ON chart_readings(chart_id);
```

**Drizzle schema in `src/shared/lib/schema.ts` (insert after `cosmicPassports` block):**
```ts
export const chartReadings = pgTable('chart_readings', {
  id: text('id').primaryKey(),
  chartId: text('chart_id')
    .notNull()
    .references(() => natalCharts.id, { onDelete: 'cascade' }),
  locale: text('locale', { enum: ['en', 'es'] }).notNull(),
  body: text('body').notNull(),
  model: text('model').notNull().default('claude-sonnet-4-20250514'),
  generatedAt: timestamp('generated_at', { withTimezone: true })
    .notNull()
    .defaultNow(),
}, (t) => ({
  uniqChartLocale: uniqueIndex('chart_readings_chart_locale_uniq').on(t.chartId, t.locale),
}));

export type ChartReading = typeof chartReadings.$inferSelect;
```

**Retention:** Reuses `natal_charts` 90-day temp-cleanup cron via `ON DELETE CASCADE`. No separate cleanup logic.

**PII:** Body contains chart-derived interpretation (sign placements, aspect descriptions) but never raw birth date, time, or location. Safe to store unencrypted per CLAUDE.md PII rules (only "birth date/time/location" is PII; derived sign data is not).

| Pro + chartId not in DB | `404 CHART_NOT_FOUND` | `errorNotFound` ("Recalculate to continue") |
| Pro + rate limit exceeded | `429 RATE_LIMITED` | `errorRateLimit` ("Try in a minute") |
| Pro + cache hit | `200 { source: 'cache' }` | Render reading instantly (no spinner) |
| Pro + cache miss + LLM ok | `200 { source: 'generated' }` | Render reading |
| Pro + Anthropic API 5xx | `502 AI_SERVICE_ERROR` | `errorGeneric` + retry button |
| Pro + `ANTHROPIC_API_KEY` missing | `503 SERVICE_UNAVAILABLE` | `errorGeneric` + Sentry alert |
| Pro + DB insert fails (cache write) | `200` (cache write is non-fatal) | Reading still renders; cache miss on next visit |
| Network fetch throw | n/a | `errorNetwork` |
