## LLM-пайплайн генерации

### Шаг 1: Системный промпт

```
Ты — профессиональный астролог, работающий в сидерической традиции.
Твои источники: Liber 777 (Crowley), Equinox Vol. I, общедоступные
астрологические принципы сидерической системы.

Стиль: глубокий, поэтичный, но доступный. Не «попсовая» астрология.
Объединяй астрологический и эзотерический контекст.

ВАЖНО: не цитируй Book of Thoth (1944, копирайт) и тексты Eshelman.
Пиши собственные формулировки на основе общедоступных принципов.
```

### Шаг 2: Генерация

- **Модель:** Claude Sonnet 4.6 (для качества) или Haiku 4.5 (для объёма)
- **Формат:** 300-500 слов на эссе
- **Стоимость:** ~$3 за MVP-корпус (120 текстов EN)
- **Структура каждого эссе:**
  1. Архетипическое значение позиции
  2. Проявление в жизни
  3. Тень / вызов
  4. Эзотерическое соответствие (из 777)
  5. Совет / медитация

### Шаг 3: Fact-Check (второй LLM-вызов)

```
Проверь этот текст на соответствие:
1. Точность астрологических соответствий (777)
2. Отсутствие цитат из защищённых источников
3. Корректность сидерических позиций
4. Отсутствие медицинских/финансовых советов

Верни: PASS / REVIEW_NEEDED с указанием проблемных мест
```

### Шаг 4: Ручная верификация

- Тексты с REVIEW_NEEDED проверяются вручную
- Все 120 текстов проходят через 1-2 практикующих астролога (beta)
- Форма обратной связи для каждого эссе

### Шаг 5: Перевод (Фаза 2)

> Переводы перенесены из MVP. Запуск на EN only, перевод после валидации PMF и данных аналитики (>10% трафика из конкретного региона).

- Claude API переводит EN → целевой язык
- Эзотерическая терминология: glossary file для consistency
- Ручная проверка первых 10 переводов, далее — выборочно

## Хранение контента

### MVP: MDX файлы в репозитории

```
src/modules/esoteric/content/
├── essays/
│   └── en/                    # MVP: EN only. Фаза 2: + es/
│       ├── sun-in-aries.mdx
│       ├── sun-in-taurus.mdx
│       └── ... (120 files)
├── signs/
│   └── en/                    # 12 files
├── planets/
│   └── en/                    # 10 files
└── houses/
    └── en/                    # 12 files
```

**Почему MDX, а не CMS:** на MVP проще. Версионирование через git. Деплой через Vercel. Нет лишней инфраструктуры.

### Фаза 2: Headless CMS (Sanity или Notion)

Когда контент начнёт обновляться чаще / появятся non-dev контрибьюторы — мигрировать в CMS. Обновление без передеплоя.

## Правила контента

### Что можно

- Собственные формулировки на основе PD-принципов
- Соответствия из 777 (public domain)
- Цитаты из Equinox Vol. I (1909-1913, PD)
- Liber AL (1909, PD)
- NASA данные и факты

### Что нельзя

- ❌ Цитаты из Book of Thoth (1944, копирайт до 2039)
- ❌ Тексты Eshelman (все защищены)
- ❌ Изображения Thoth Tarot (Frieda Harris, до 2064)
- ❌ Медицинские/финансовые советы
- ❌ Фразы вроде «звёзды предсказывают» или «вам суждено»

### Editorial Style Guide

См. `docs/editorial-style-guide.md` — тон, стиль, запрещённые формулировки, примеры.

### Disclaimer (в каждом эссе)

```
Астрологическая интерпретация не является медицинским, финансовым
или психологическим советом. Это инструмент самопознания и
рефлексии. Расчёты: Swiss Ephemeris | Ayanamsa: Lahiri.
```
