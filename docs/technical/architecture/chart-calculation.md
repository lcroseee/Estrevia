# Расчёт натальной карты: от ввода до экрана

> Самый важный техпроцесс в Estrevia. Ошибка здесь = смерть продукта.

---

## Простыми словами

Натальная карта — это «фотография неба» в момент рождения. Мы берём дату, время и место → вычисляем, где стояла каждая планета → показываем на круговой диаграмме (колесо).

Сидерическая карта отличается от тропической на ~24° (прецессия равноденствий). Это значит, что у многих людей знаки «сдвигаются» на один назад.

---

## Шаг за шагом

### Шаг 1: Ввод данных

```
Пользователь вводит:
  ├── Дата: 15 марта 1990
  ├── Время: 14:30 (опционально)
  └── Город: Moscow, Russia

Что нужно получить:
  ├── Географические координаты: lat 55.7558, lon 37.6173
  ├── Timezone на момент рождения: Europe/Moscow (UTC+3 в 1990)
  └── Julian Day Number: 2447965.97917
```

**Geocoding города:** предзагруженный JSON со списком ~50K городов (lat/lon/timezone). Автокомплит при вводе. Не Google Maps API — никаких внешних запросов.

**Timezone:** библиотека `timezone-lookup` — по координатам + дате определяет timezone. Важно: timezone может меняться исторически (Россия менял UTC offset несколько раз).

**Julian Day:** астрономический формат даты — один непрерывный номер дня с 1 января 4713 BC. Swiss Ephemeris работает с Julian Day.

```
JD = date_to_jd(1990, 3, 15, 14, 30, 0, timezone_offset)
   = 2447965.97917
```

### Шаг 2: Swiss Ephemeris WASM

```
Входные данные:
  ├── julian_day: 2447965.97917
  ├── planet_id: SE_SUN (0), SE_MOON (1), ... SE_PLUTO (9), ...
  └── flags: SEFLG_SWIEPH | SEFLG_SPEED

Для каждой из 12 небесных тел вызываем:
  swe_calc_ut(julian_day, planet_id, flags)

Результат для каждого тела:
  [0] ecliptic longitude  = 354.254°  (положение на эклиптике)
  [1] ecliptic latitude   = -0.0001°  (отклонение от эклиптики)
  [2] distance            = 0.9935 AU (расстояние от Земли)
  [3] speed in longitude  = 1.0194°/day (скорость движения)
```

**12 тел:** Солнце, Луна, Меркурий, Венера, Марс, Юпитер, Сатурн, Уран, Нептун, Плутон, Северный Лунный Узел, Хирон.

**Время вычисления:** < 50ms для всех 12 тел. Один вызов swe_calc_ut ~ 2-4ms.

### Шаг 3: Тропическая → Сидерическая конверсия

```
Тропическая позиция (из Swiss Ephemeris):
  Солнце = 354.254° (24°15' Рыб тропически)

Ayanamsa (сдвиг) для 1990 года:
  Lahiri:          23.732°
  Fagan-Bradley:   24.736°
  Krishnamurti:    23.828°

Сидерическая позиция (Lahiri):
  354.254° - 23.732° = 330.522° (0°31' Рыб сидерически)

Или напрямую через Swiss Ephemeris:
  swe_set_sid_mode(SE_SIDM_LAHIRI)
  → автоматический пересчёт
```

**Toggle tropical/sidereal:** пересчёт = вычитание одного числа (ayanamsa). Мгновенно, < 1ms. Не нужен повторный вызов Swiss Ephemeris.

### Шаг 4: Знак, градус, минута

```
Абсолютные градусы → знак + позиция в знаке:

  longitude = 330.522°
  sign_index = floor(330.522 / 30) = 11  → Pisces (♓)
  degree_in_sign = 330.522 mod 30 = 0.522°
  
  0.522° = 0° 31' 19.2"
           ↑    ↑    ↑
        градус минуты секунды
  
  Итог: Солнце в 0°31' Рыб (сидерически)
         vs  24°15' Рыб (тропически)
```

### Шаг 5: Дома (если время известно)

```
Если время рождения указано:
  swe_houses_ex(julian_day, lat, lon, 'P')
  // 'P' = Placidus system
  
  → 12 cusps (начала домов):
    House 1 (ASC): 182.5° → 2°30' Весы
    House 2: 210.3° → 0°18' Скорпион
    House 7 (DSC): 2.5° → 2°30' Овен
    House 10 (MC): 92.1° → 2°06' Рак
    ...

Если время НЕ указано:
  → Дома = null
  → ASC = null
  → В UI: "Укажите время рождения для расчёта домов и восходящего знака"
```

### Шаг 6: Аспекты

```
Для каждой пары планет (66 пар из 12):
  diff = |planet1.longitude - planet2.longitude|
  if diff > 180: diff = 360 - diff

Проверяем каждый тип аспекта:
  ┌─────────────┬───────────┬────────────────┐
  │ Аспект      │ Угол      │ Допуск (орбис) │
  ├─────────────┼───────────┼────────────────┤
  │ Conjunction  │ 0°        │ ±8°            │
  │ Opposition   │ 180°      │ ±8°            │
  │ Trine        │ 120°      │ ±8°            │
  │ Square       │ 90°       │ ±7°            │
  │ Sextile      │ 60°       │ ±6°            │
  │ Quincunx     │ 150°      │ ±3°            │
  │ Semi-sextile │ 30°       │ ±2°            │
  └─────────────┴───────────┴────────────────┘

Пример:
  Солнце = 330.5°, Марс = 242.3°
  diff = |330.5 - 242.3| = 88.2°
  Ближайший аспект: Square (90°)
  Орбис: |90 - 88.2| = 1.8° (< 7° допуска)
  → Square с орбисом 1.8° ✓

Applying vs Separating:
  Если быстрая планета приближается к точному аспекту → applying
  Если удаляется → separating
  (определяется по speed обеих планет)
```

### Шаг 7: Рендеринг SVG колеса

```
Из данных → визуализация:

                    ♈ 0°
                   /    \
              ♓ /        \ ♉
              /            \
         ♒ /    Планеты     \ ♊
         |     на своих      |
    ♑ ── |    градусах в     | ── ♋
         |     кольце        |
         ♐ \                / ♌
              \            /
              ♏ \        / ♍
                   \    /
                    ♎ 180°

SVG элементы:
  1. Внешнее кольцо: 12 секторов по 30° (знаки)
  2. Среднее кольцо: 12 секторов (дома, если известны)
  3. Планеты: точки на соответствующих градусах
  4. Линии аспектов: соединяют пары планет
     - Красные: квадрат, оппозиция (напряжённые)
     - Синие: трин, секстиль (гармоничные)
     - Зелёные: конъюнкция
  5. Глифы: ☉☽☿♀♂♃♄♅♆♇ на позициях планет
```

## Performance Budget

| Операция | Цель | Реальность |
|----------|------|-----------|
| Geocode города | < 100ms | ~10ms (JSON lookup) |
| Timezone lookup | < 50ms | ~5ms |
| Swiss Eph: 12 тел | < 100ms | ~30-50ms |
| Ayanamsa пересчёт | < 1ms | < 0.1ms |
| Аспекты (66 пар) | < 10ms | ~2ms |
| SVG рендеринг | < 200ms | ~100ms |
| **Итого: ввод → карта** | **< 500ms** | **~200ms** |
