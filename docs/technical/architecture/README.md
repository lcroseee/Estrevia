### Ключевая идея

**Расчёт карты = на сервере.** Swiss Ephemeris работает через `sweph` (Node.js native C bindings) на Vercel Functions. Клиент отправляет данные рождения → сервер вычисляет позиции → клиент рендерит SVG. Это даёт:

- **Проверенную стабильность** (нативные биндинги, не экспериментальный WASM)
- **Простой деплой** (`npm install sweph`, Vercel Fluid Compute переиспользует инстансы)
- **Консистентные результаты** (одинаковый расчёт на всех устройствах)
- **Быстрый отклик** (< 50ms расчёт + сетевая задержка, Fluid Compute минимизирует cold start)

Сервер отвечает за: расчёт карт, сохранение карт, аутентификацию, NASA/USGS данные, подписки, аналитику.

### Правила связей

```
✅ Модуль → Shared          (все используют общие типы)
✅ App Router → Модуль       (страницы собирают модули)
❌ Модуль → Модуль           (нет прямых зависимостей!)
❌ Shared → Модуль           (shared ничего не знает о модулях)
```

**Почему нет связей между модулями:** если astro-engine зависит от data-feed — их нельзя будет разделить потом. Связь идёт через типы в `shared/` и через UI на уровне страницы.

## Что обрабатывает клиент vs сервер

| Действие | Где происходит | Почему |
|----------|---------------|--------|
| Расчёт натальной карты | **Сервер** (sweph API) | Стабильность, консистентность, проверенные нативные биндинги |
| Расчёт лунной позиции | **Сервер** (sweph API) | Точность, единый движок расчёта |
| Расчёт планетарных часов | **Сервер** (sweph API) | Точность восхода/захода, единый движок |
| Переключение sidereal/tropical | **Клиент** (offset) | Простое вычитание ayanamsa (~24°), не требует сервера |
| Отображение эссе | **Клиент** (кэш/MDX) | Статичный контент, ISR + CDN кэш |
| Сохранение карты | **Сервер** (API) | Нужна БД + auth |
| Регистрация / логин | **Сервер** (Clerk) | Auth = серверная логика |
| NASA/USGS данные | **Сервер** (Cron → Redis) | Один запрос к NASA на всех юзеров, не один на каждого |
| Оплата подписки | **Сервер** (Stripe) | PCI compliance, webhooks |
| Аналитика | **Клиент** (PostHog JS) | Events отправляются из браузера |
| Share card (PNG) | **Сервер** (OG Image) | next/og генерирует картинку на сервере |
