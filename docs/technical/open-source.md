# Open Source стратегия

## Решение

**Estrevia выпускается как open source под лицензией AGPL-3.0.**

---

## Почему open source

### 1. Решает лицензию Swiss Ephemeris ($0 вместо ???)

Swiss Ephemeris — AGPL. Если наш код закрыт — нужна коммерческая лицензия (цена неизвестна, переговоры с Astrodienst). Если код открыт — AGPL выполнена автоматически. Проблема исчезает.

### 2. Код — не moat

| Что можно скопировать | Что нельзя скопировать |
|----------------------|----------------------|
| Next.js приложение | 240 эзотерических эссе |
| Swiss Ephemeris расчёты | 50K+ зарегистрированных пользователей |
| UI компоненты | Бренд ESTREVIA (USPTO) |
| API маршруты | Community и word-of-mouth |
| Prisma-схема | Данные пользователей |
| PostHog интеграция | Organic SEO (120 проиндексированных страниц) |

Co-Star может открыть весь код завтра — никто не воспроизведёт 20M юзеров и бренд. Код = кирпичи. Продукт = здание + жильцы.

### 3. Доверие целевой аудитории

- **Thelema/оккультное сообщество** ценит прозрачность и открытость. «93» = делай что пожелаешь. Закрытый код = подозрение
- **Сидерические астрологи** — дотошные. Они **хотят** видеть, как считается карта. Открытый код = доверие к расчётам
- **Астрономы** (сегмент crossover) — уважают open source. «Swiss Ephemeris, AGPL, code on GitHub» — разговор на их языке
- **Анти-алгоритм** позиционирование усиливается: «Мы не прячем ничего. Вот наш код»

### 4. Практические выгоды

| Выгода | Как работает |
|--------|-------------|
| **Контрибуции** | Астрологи присылают PR с исправлениями расчётов. Бесплатный QA от экспертов |
| **PR / маркетинг** | «Open source astrology platform» — виральный заголовок для HackerNews, Product Hunt |
| **Рекрутинг** | Ко-фаундер смотрит код до разговора. Показывает quality и серьёзность |
| **Безопасность** | Community находит уязвимости быстрее, чем один разработчик |
| **Long-term trust** | Если Estrevia закроется — community может продолжить. Пользователи это ценят |

## Что открываем, что нет

### ✅ Open source (AGPL-3.0)

| Компонент | Почему открываем |
|-----------|-----------------|
| Весь код приложения (Next.js) | AGPL compliance. Код не moat |
| Астро-движок (Swiss Ephemeris WASM) | AGPL требует |
| UI компоненты (колесо, лунный календарь) | Community contributions |
| API routes | Прозрачность |
| Prisma-схема | Прозрачность |
| CI/CD конфигурация | Помогает контрибуторам |
| Тесты (100+ эталонных карт) | Верификация точности community |

### ❌ Не открываем (proprietary)

| Компонент | Почему закрываем | Защита |
|-----------|-----------------|--------|
| **240 эссе** (контент) | Интеллектуальная собственность. Стоимость: время + Claude API + верификация | Лицензия: CC BY-NC-ND или proprietary |
| **Данные пользователей** | PII, GDPR, очевидно | Шифрование, access control |
| **Бренд / логотип / дизайн** | Идентичность | USPTO trademark |
| **Промпты для Claude** | Competitive advantage в качестве контента | Можно пересмотреть позже |
| **Коммерческая инфраструктура** | Stripe keys, Clerk config, env vars | .env, Vercel secrets |

### ⚠️ На усмотрение

| Компонент | Аргумент за открытие | Аргумент против |
|-----------|---------------------|-----------------|
| Промпты для Claude | Повышает trust, community улучшает | Конкурент может сгенерировать аналогичный контент |
| Дизайн-токены (цвета, шрифты) | Помогает контрибуторам | Снижает визуальную уникальность форков |

## Лицензия: AGPL-3.0

### Почему AGPL, а не MIT/Apache

| Лицензия | Разрешает closed fork? | Для нас |
|----------|----------------------|---------|
| **MIT** | ✅ Да — кто угодно может закрыть | ❌ Конкурент берёт код, закрывает, не контрибутит обратно |
| **Apache 2.0** | ✅ Да | ❌ Та же проблема |
| **GPL-3.0** | ❌ Форки должны быть GPL | ⚠️ Но SaaS лазейка: можно запустить как сервис не раскрывая |
| **AGPL-3.0** | ❌ Даже SaaS форки должны быть открыты | ✅ **Полная защита** |

AGPL-3.0 = «используй как хочешь, но если модифицируешь и запускаешь как сервис — открой свой код». Это стандарт для open source SaaS: PostHog, Cal.com, Mattermost.

### Как это работает для пользователей

- **Обычные пользователи** (estrevia.app): ничего не меняется. Используют как сервис
- **Self-hosters**: могут развернуть у себя бесплатно. Мы даже поможем (docker-compose в repo)
- **Форкеры-конкуренты**: обязаны открыть свои изменения. AGPL = protection
- **Контрибуторы**: присылают PR, мы мержим. Стандартный CLA (Contributor License Agreement)

## Успешные примеры

| Продукт | Лицензия | Revenue | Модель |
|---------|----------|---------|--------|
| **PostHog** | MIT → переход на proprietary для enterprise | $100M+ оценка | Open core + cloud |
| **GitLab** | MIT (CE) + proprietary (EE) | $500M+ ARR | Open core |
| **Cal.com** | AGPL-3.0 | $30M+ funding | Open source + cloud |
| **Plausible** | AGPL-3.0 | Прибыльны, bootstrap | Open source + cloud |
| **Mattermost** | AGPL-3.0 (community) | $50M ARR | Open core |

### Их формула

```
Код = бесплатный, открытый
Хостинг = платный (cloud version)
Enterprise фичи = платные (если нужны)
Community = бесплатное, самоподдерживающееся
```

### Формула Estrevia

```
Код = AGPL-3.0, открытый
Контент (эссе) = proprietary (CC BY-NC-ND)
Хостинг estrevia.app = платный (Star/Cosmos подписки)
Self-host = бесплатный (но без эссе контента)
Community = бесплатное на estrevia.app
```

## Риски open source

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| **Конкурент форкнет** | Средняя | AGPL: форк тоже должен быть открытым. Плюс: наш moat — контент + community |
| **Self-hosters не платят** | Высокая | ОК. Self-host версия = бесплатный маркетинг. Кто self-hostит — не наша ЦА (не заплатил бы и так) |
| **Плохие PR / вандализм** | Низкая | Code review обязателен. Branch protection. CLA |
| **Раскрытие уязвимостей** | Низкая | Responsible disclosure policy. Security.md в repo |
| **«Зачем платить если код открыт?»** | Средняя | Платят за контент (240 эссе), hosted service, community — не за код |
