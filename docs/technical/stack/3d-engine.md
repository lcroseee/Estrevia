# 3D движок: Three.js + React Three Fiber

> Фаза 2-3. Не для MVP, но архитектурно важно заложить.

## Что это простыми словами

**Three.js** — библиотека для 3D-графики в браузере. Планеты, звёзды, созвездия, солнечная система — всё через Three.js. Как Unity, но для веба, бесплатно.

**React Three Fiber (R3F)** — обёртка Three.js для React. Вместо императивного кода `scene.add(mesh)` пишешь `<mesh />` как React-компонент. Реактивное обновление, интеграция с React-состоянием.

---

## Почему Three.js + R3F

| Причина | Подробности |
|---------|------------|
| **Доминирование** | 270x больше npm downloads чем Babylon.js. Абсолютный стандарт |
| **React-native** | R3F = 3D как React-компоненты. Один mental model |
| **Экосистема** | Drei (helpers), r3f-perf (profiling), zustand (state) |
| **NASA текстуры** | Public domain текстуры планет → на Three.js сферы. Бесплатно |
| **WebGPU** | Safari 26+ (сентябрь 2025) = cross-browser WebGPU. До 10x performance |

## Альтернативы

| Движок | Плюсы | Минусы |
|--------|-------|--------|
| **Babylon.js** | Больше встроенного (физика, GUI, XR) | Нет React-интеграции уровня R3F. Тяжелее |
| **PlayCanvas** | Визуальный редактор | Менее code-first. Меньше сообщество |
| **Spline** | Drag-and-drop 3D дизайн | Для маркетинга, не для programmatic сцен |

---

## Вердикт

**Three.js + R3F — безальтернативный выбор** для программируемого 3D в React-приложении. Экосистема, сообщество, performance — всё на стороне Three.js. WebGPU добавляет future-proofing. На MVP — не нужен, но architecture-ready.
