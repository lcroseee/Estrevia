# Стили: Tailwind CSS 4 + shadcn/ui

## Что это простыми словами

**Tailwind CSS** — это набор маленьких CSS-классов. Вместо того чтобы писать `color: white; padding: 16px; background: black;` в отдельном файле, ты пишешь `text-white p-4 bg-black` прямо в HTML. Звучит странно, но это в 3-5x быстрее для разработки.

**shadcn/ui** — это НЕ библиотека компонентов (как Material UI). Это **коллекция кода**, который ты копируешь в свой проект. Кнопка, модалка, табы, календарь — всё с accessibility из коробки. Ты владеешь кодом и можешь менять всё.

---

## Почему Tailwind CSS

| Причина | Для Estrevia |
|---------|-------------|
| **Тёмная тема за 5 минут** | `dark:bg-[#0A0A0F] dark:text-white` — космический минимализм из коробки |
| **Скорость** | v4: Rust-движок, полная пересборка < 100ms (было 3.5s в v3) |
| **Один файл** | Стили прямо в компоненте. Нет переключения между .tsx и .css |
| **Responsive** | `md:grid-cols-2 lg:grid-cols-3` — адаптивность в одну строку |
| **shadcn/ui** | Лучшая библиотека компонентов для React написана на Tailwind |

### Что изменилось в v4

| Было (v3) | Стало (v4) |
|-----------|-----------|
| `tailwind.config.js` (JavaScript) | `@theme` в CSS (нет JS-файла) |
| PostCSS plugin | Standalone инструмент |
| 3.5s полная пересборка | < 100ms (Rust-движок Oxide) |
| `bg-gradient-to-r` | `bg-linear-to-*` |
| `flex-shrink-0` | Переименован |

**Миграция:** автотул `npx @tailwindcss/upgrade` делает ~80% работы. Остальное — 1-3 часа ручной правки.

## Почему shadcn/ui

| Причина | Подробности |
|---------|------------|
| **Копируемый код** | Не `npm install` — а `npx shadcn add button`. Код в твоём проекте, полный контроль |
| **Accessibility** | Построен на Radix UI (теперь + Base UI). ARIA, keyboard nav, screen readers — из коробки |
| **Tailwind-native** | Идеальная интеграция. Кастомизация через CSS variables |
| **Next.js стандарт** | De-facto выбор для Next.js проектов в 2025-2026 |

### Что мы берём из shadcn/ui

| Компонент | Для чего |
|-----------|----------|
| Button, Card, Dialog | Базовый UI |
| Tabs | Sidereal/Tropical toggle |
| Calendar | Лунный календарь |
| Tooltip | Подсказки по планетам |
| Navigation Menu | Навигация |
| Form + Input | Ввод даты рождения |
| Toast | Уведомления |

### Что делаем custom (не shadcn)

| Компонент | Почему custom |
|-----------|-------------|
| **Chart Wheel** | SVG натальная карта — уникальный UI, нет аналогов |
| **Planet Icon** | Астрологические глифы + цвета + анимация |
| **Moon Phase** | SVG с тенью, real-time отображение |
| **Planetary Hours** | Кольцевая диаграмма real-time |

## Альтернативы: почему не они

### CSS Modules / Vanilla CSS

| За | Против |
|----|--------|
| Знакомо всем | В 3-5x медленнее разработка. Куча файлов |
| Нет зависимости | Нет design system из коробки |
| Чистый HTML | Responsive, тёмная тема — всё вручную |

### Material UI (MUI)

| За | Против |
|----|--------|
| Огромная библиотека | Material Design = Google look. Не «космический минимализм» |
| Зрелая, протестированная | Bundle size большой. Runtime CSS-in-JS |
| | Кастомизация — борьба с фреймворком |

### Styled Components / Emotion

| За | Против |
|----|--------|
| CSS-in-JS, привычно | Runtime overhead. Не совместимо с Server Components |
| | Next.js App Router плохо работает с runtime CSS-in-JS |
| | Тренд уходит: Tailwind + shadcn побеждает в экосистеме |

## Подводные камни

### 1. «Все shadcn проекты выглядят одинаково»

**Проблема:** дефолтная тема = серый/белый минимализм. Половина Next.js проектов выглядят идентично.

**Решение:** наша цветовая палитра (Deep Space #0A0A0F, планетарные цвета, золото/серебро) + кастомные шрифты (Geist + Crimson Pro) сделают Estrevia непохожей.

### 2. Tailwind v4 ломает плагины

~40% плагинов от v3 не работают в v4 (функциональность переехала в ядро).

**Решение:** начинаем сразу на v4. Нет legacy плагинов — нет проблем.

### 3. Radix UI замедлился после покупки WorkOS

shadcn/ui зависит от Radix. Если Radix перестанет обновляться — проблема.

**Решение:** shadcn/ui добавил поддержку Base UI (MUI) как альтернативного слоя. Плюс мы владеем кодом — можем заменить примитивы вручную.

## Вердикт

**Tailwind 4 + shadcn/ui = стандарт индустрии для Next.js.** Быстрая разработка, accessibility из коробки, полный контроль над кодом. Космическая тема поверх — через CSS variables и кастомные компоненты (Chart Wheel, Planet Icons).
