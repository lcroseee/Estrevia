# Аутентификация: Clerk

## Что это простыми словами

Clerk — это сервис, который берёт на себя **всё** связанное с логином: регистрация, вход, Google/Apple OAuth, magic links, 2FA, управление сессиями, компоненты UI (готовые формы). Вместо того чтобы строить auth с нуля (2-4 недели), подключаешь Clerk за 2 часа.

---

## Почему Clerk

| Причина | Для Estrevia |
|---------|-------------|
| **Скорость** | Auth за 2 часа вместо 2 недель. На MVP — критично |
| **Next.js интеграция** | Лучшая в индустрии. `<SignIn />` компонент, middleware, server-side auth |
| **OAuth** | Google, Apple, GitHub — из коробки. Magic links — из коробки |
| **Vercel Marketplace** | Нативная интеграция, env variables автоматически |
| **UI компоненты** | Готовые формы входа/регистрации. Кастомизируемые |
| **Free tier** | 10,000 MAU бесплатно. Достаточно для MVP + первые месяцы |

## ⚠️ Главная проблема: цена при росте

| MAU | Стоимость Clerk | В месяц |
|-----|----------------|---------|
| 1,000 | Free | $0 |
| 10,000 | Free | $0 |
| 10,001 | Pro ($0.02/MAU сверх 10K) | $0.02 |
| 50,000 | Pro | **$800/мес** |
| 100,000 | Pro | **$1,800/мес** |

**$800/мес за auth при 50K пользователях — дорого.** При MRR $20K (месяц 9 по нашему плану) — это 4% от выручки только на auth.

### Что пишут

- «Clerk — лучший DX. Но когда растёшь, цена становится проблемой»
- «MFA добавляет $100/мес внезапно — нигде не написано явно»
- «Миграция с Clerk сложная — пользователи привязаны к их user IDs»

## Альтернативы

### Auth.js (бывший NextAuth v5)

| Аспект | Подробности |
|--------|------------|
| **Цена** | $0 навсегда. Open source |
| **Плюсы** | Бесплатно, огромное сообщество, полный контроль |
| **Минусы** | Нет готовых UI компонентов. Нет 2FA, passkeys, organizations из коробки. v5 долго в beta. Нужно строить всё самому |
| **Время на настройку** | 1-2 недели (вместо 2 часов) |
| **Когда использовать** | Если бюджет $0 и есть время на разработку |

### Supabase Auth

| Аспект | Подробности |
|--------|------------|
| **Цена** | 50K MAU бесплатно (!). Потом ~$0.00325/MAU |
| **Плюсы** | В 5x дешевле Clerk при росте. Интеграция с Supabase DB (Row Level Security) |
| **Минусы** | Привязывает к Supabase экосистеме. Меньше готовых UI компонентов. Next.js интеграция слабее |
| **Когда использовать** | Если используешь Supabase как основную БД |

### Better Auth (новичок 2024-2026)

| Аспект | Подробности |
|--------|------------|
| **Цена** | $0 навсегда. Open source. 20K+ GitHub stars за 1.5 года |
| **Плюсы** | Full-featured (2FA, passkeys, RBAC, impersonation). Нет vendor lock-in. $20-50/мес total cost при любом количестве пользователей |
| **Минусы** | Молодой проект (с сентября 2024). Меньше documentation и community |
| **Когда использовать** | Если готов к менее зрелому инструменту ради полного контроля и $0 стоимости |

## Стратегия для Estrevia

### MVP (месяцы 1-6): Clerk

- Быстрый старт
- 10K MAU бесплатно = хватит на MVP + первые месяцы после запуска
- Не тратить время на auth когда нужно строить астро-движок

### При достижении 10K MAU: оценить миграцию

| Путь | Условие |
|------|---------|
| Остаться на Clerk | MRR >> стоимость Clerk. $800/мес auth приемлемо |
| Мигрировать на Better Auth | MRR < $5K. Нужно экономить. Better Auth к тому моменту будет зрелее |
| Мигрировать на Auth.js | Нужен полный контроль и кастомные auth flows |

### Как подготовиться к миграции

1. **Не использовать Clerk user ID напрямую.** Хранить свой UUID в таблице User, маппить на `clerk_id`
2. **Абстрагировать auth в модуль.** `src/modules/auth/` — все вызовы Clerk через обёртку. При миграции меняем обёртку, не весь код
3. **Не полагаться на Clerk-specific фичи** (Organizations, RBAC) без необходимости

---

## Вердикт

**Clerk для MVP = правильно.** Быстро, работает, бесплатно до 10K. Но это **осознанно временное решение** — при росте цена станет проблемой. Закладываем абстракцию с первого дня.
