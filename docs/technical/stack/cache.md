# Кэш: Upstash Redis

## Что это простыми словами

Redis — это сверхбыстрое хранилище «в памяти». Работает в 100x быстрее базы данных, потому что данные не на диске, а в оперативной памяти. Используем для: кэша NASA-данных (чтобы не дёргать API каждый запрос), rate limiting (защита от спама), и временных данных.

**Upstash** — это Redis в облаке с HTTP API. Ключевая особенность: работает из serverless функций (Vercel) без TCP-соединений. Pay-per-use: платишь за команды, а не за время работы сервера.

---

## Почему Upstash Redis

| Причина | Для Estrevia |
|---------|-------------|
| **HTTP API** | Работает из Vercel Functions / Edge — нет проблем с TCP connections |
| **Scale-to-zero** | Нет трафика = $0. Идеально для MVP |
| **Vercel native** | Vercel KV построен на Upstash. Нативная интеграция |
| **Rate limiting** | `@upstash/ratelimit` — защита API за 5 строк кода |
| **Free tier** | 10K commands/день. Для MVP хватит |

## Для чего конкретно

| Задача | Как используем | TTL |
|--------|---------------|-----|
| **Кэш NASA DONKI** | Солнечные вспышки → Redis → API отдаёт из кэша | 5 мин |
| **Кэш USGS** | Землетрясения → Redis | 5 мин |
| **Rate limiting** | POST /api/chart/calculate: 10 req/мин (guest) | — |
| **Лунные позиции** | Текущая фаза Луны, пересчёт раз в час | 1 час |
| **Планетарные часы** | Текущий управитель, пересчёт каждые 5 мин | 5 мин |

---

## Альтернативы

| Сервис | Плюсы | Минусы | Цена |
|--------|-------|--------|------|
| **Redis Cloud** | Дешевле при высоких объёмах (10M+ req) | TCP-only — проблема для serverless | $7+/мес |
| **Momento** | 50GB free transfer | Меньше экосистема, новый | Pay-per-use |
| **Vercel KV** | Самая простая интеграция | 2x дороже Upstash (построен на нём!) | Pay-per-use |

**Почему не Vercel KV:** буквально тот же Upstash, но с наценкой. Подключаем Upstash напрямую — дешевле.

---

## Вердикт

**Upstash — очевидный выбор для Vercel + serverless.** HTTP API, scale-to-zero, бесплатный старт. Нет причин выбирать что-то другое на данном этапе.
