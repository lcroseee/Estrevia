### ❌ Errors (3) — manual founder action required

#### 1. META_CAPI_TOKEN not set

**Action:** Generate Conversions API Access Token

1. Open `business.facebook.com/events_manager`
2. Select your Pixel
3. **Settings → Conversions API → Generate Access Token**
4. Add to `.env`: `META_CAPI_TOKEN=<token>`

#### 2. Meta Ad Account 403

```
HTTP 403 — "(#200) Ad account owner has NOT grant ads_management or ads_read permission"
```

**Cause:** System User token works (Meta `/me` succeeded with app_id), but the System User has no permissions on the specific Ad Account.

**Action:** Grant System User permissions on Ad Account

1. `business.facebook.com/settings`
2. **Users → System Users → click your Estrevia agent System User**
3. **Add Assets → Ad Accounts**
4. Select the ad account `act_<id>`
5. Enable **Manage Ads** (or Manage Campaigns)
6. Save

After this, re-run `npm run advertising:pre-launch-check` to verify.

#### 3. CAPI test event skipped

Depends on resolving #1 above.
