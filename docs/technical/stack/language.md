# Язык: TypeScript

## Что это простыми словами

TypeScript — это JavaScript с «проверкой типов». Представь, что JavaScript позволяет положить строку «hello» туда, где ожидается число. И ты узнаёшь об ошибке только когда пользователь видит NaN на экране. TypeScript ловит такие ошибки **до запуска** — прямо в редакторе.

---

## Почему TypeScript

| Причина | Для Estrevia |
|---------|-------------|
| **Астрономическая точность** | Градусы, минуты, секунды дуги, ayanamsa — ошибка в типе = ошибка в расчёте = потеря доверия |
| **Один язык** | Frontend, backend, API routes, утилиты — всё на одном языке. Один человек может делать всё |
| **Автодополнение** | Drizzle-схема → типы БД → типы API → типы UI. Цепочка типов от базы до экрана |
| **Рефакторинг** | Переименовал поле `birthDate` → TypeScript покажет все 47 мест, где нужно поправить |
| **Стандарт** | Next.js, Drizzle, Clerk, shadcn/ui — все написаны на TypeScript. Работают лучше с TS |

---

## Почему не чистый JavaScript

| Аргумент «за JS» | Контраргумент |
|-------------------|--------------|
| «Быстрее начать» | На 10 минут быстрее. На 10 часов дольше дебажить |
| «Проще для новичков» | Estrevia = астрономические расчёты с точностью ±0.01°. Нужна строгость |
| «Нет compile step» | Next.js и так компилирует. TypeScript добавляет 0 секунд к билду |
| «Типы мешают» | `strict: true` с первого дня. Боль сейчас → спокойствие потом |

## Конкретный пример: почему это критично для Estrevia

```typescript
// Без TypeScript — баг обнаружен через 3 месяца:
function calculatePosition(date, lat, lon, ayanamsa) {
  // Кто-то передал lat и lon в градусах, а функция ожидает радианы
  // Результат: ошибка в 57x. Карта показывает бред.
  // Никакого предупреждения.
}

// С TypeScript — баг обнаружен мгновенно:
function calculatePosition(
  date: Date,
  lat: Degrees,    // branded type
  lon: Degrees,    // branded type
  ayanamsa: 'lahiri' | 'fagan_bradley' | 'krishnamurti'
): PlanetaryPosition {
  // TypeScript не даст передать радианы вместо градусов
  // TypeScript не даст передать строку вместо Date
  // TypeScript не даст написать 'lahiry' (опечатка)
}
```

---

## Настройка

```json
// tsconfig.json — строгий режим с первого дня
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true
  }
}
```

**Не** включать `any` нигде. Если что-то сложно типизировать — это сигнал, что архитектура нуждается в пересмотре.
