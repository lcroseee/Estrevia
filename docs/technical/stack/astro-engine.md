# Астро-движок: Swiss Ephemeris

## Что это простыми словами

Swiss Ephemeris — это программа, которая знает **где находится каждая планета** в любой момент за последние 13,000 лет. Создана швейцарской компанией Astrodienst (те же люди, что делают Astro.com) на основе данных NASA JPL. Это **золотой стандарт** — 99% астрологических программ в мире используют Swiss Ephemeris.

Для Estrevia: пользователь вводит дату рождения → Swiss Ephemeris вычисляет положение Солнца, Луны и всех планет в градусах → мы показываем натальную карту.

---

## Почему Swiss Ephemeris

| Причина | Подробности |
|---------|------------|
| **Точность** | ±0.001 угловой секунды. Для нас допуск ±0.01° — с огромным запасом |
| **Диапазон** | 12,899 BC — 16,900 AD. Любая дата рождения |
| **Стандарт** | Astro.com, Solar Fire, Jagannatha Hora, AstroSage — все на Swiss Ephemeris |
| **Бесплатно** | GPL лицензия (с оговорками — см. ниже) |
| **Sidereal** | Встроенная поддержка ayanamsa: Lahiri, Fagan-Bradley, Krishnamurti и 30+ других |
| **Node.js** | Готовые нативные биндинги (`sweph`) для серверного расчёта |

## ⚠️ КРИТИЧНО: Лицензия

### Проблема

Swiss Ephemeris использует **двойное лицензирование:**

| Лицензия | Что значит |
|----------|-----------|
| **AGPL** | Если твоё приложение доступно через интернет (SaaS, web app) — **весь исходный код должен быть открытым под AGPL** |
| **Коммерческая** | Платная лицензия от Astrodienst. Цена не публична — нужно договариваться |

**Это значит:** если Estrevia использует Swiss Ephemeris и не открывает код — нужна коммерческая лицензия.

### Решение: Open Source (AGPL-3.0)

**Estrevia выпускается как open source под AGPL-3.0.** Это полностью решает вопрос лицензии Swiss Ephemeris:

- AGPL Swiss Ephemeris требует, чтобы код SaaS был открыт — **мы и так открываем**
- Коммерческая лицензия не нужна → $0, без переговоров
- AGPL для нашего кода **защищает** от закрытых форков: если кто-то модифицирует и запустит как SaaS — обязан открыть свои изменения
- Moat = не код, а контент (240 эссе), данные, бренд, community

Подробнее о стратегии open source: [open-source.md](../open-source.md)

## npm-пакеты

| Пакет | Тип | Лицензия | Статус |
|-------|-----|---------|--------|
| **`sweph` (timotejroiko)** | **Native N-API** | **AGPL (2.10.1+)** | **Выбран — самый поддерживаемый** |
| `swisseph` | Native wrapper | GPL | Менее активный |
| `swisseph-wasm` (prolaxu) | WASM | GPL-3.0 | Новый, маленький, не production-ready |

### Серверный расчёт (выбранный подход)

```
Пользователь вводит дату →
  POST /api/chart/calculate →
    Сервер (Vercel Function): sweph Node.js native (< 50ms) →
      JSON ответ: позиции всех планет в градусах →
        Клиент: рендеринг SVG колеса
```

**Почему серверный, а не WASM:**
- `sweph` — проверенный пакет с нативными биндингами, `npm install` и всё работает
- WASM-вариант (`swisseph-wasm`) — экспериментальный, мало пользователей, непроверен
- Серверный расчёт — стандартный подход (Astro.com, Co-Star, все конкуренты)
- Vercel Fluid Compute переиспользует инстансы — нет cold start на каждый запрос
- Не нужно грузить ~2MB WASM в браузер

## Альтернативы: почему не они

### Astronomia / astronomical-almanac (JS)

| За | Против |
|----|--------|
| Чистый JavaScript, нет WASM | Точность значительно ниже (±0.1° для некоторых планет) |
| MIT лицензия | Нет сидерической поддержки из коробки |
| | Не используется профессиональными астрологами = нет доверия |

### Свой расчёт по формулам

| За | Против |
|----|--------|
| Полный контроль, нет лицензии | Месяцы разработки. Planetary perturbations = rocket science |
| | Ошибки = смерть продукта. Астрологи проверяют до третьего знака |
| | Зачем изобретать то, что Swiss Ephemeris делает 30 лет |

### Pre-computed ephemeris (таблица)

| За | Против |
|----|--------|
| Нет runtime лицензии | Огромный объём данных (каждая минута × 10 планет × 100 лет) |
| Мгновенный lookup | Интерполяция между точками = потеря точности |
| | Нет гибкости для ayanamsa переключения |

## Вердикт

**Swiss Ephemeris — единственный реальный выбор.** Альтернативы либо недостаточно точны, либо требуют лет разработки. Используем `sweph` (Node.js native bindings) на сервере. Код Estrevia открыт под AGPL-3.0, что решает вопрос лицензии Swiss Ephemeris.

> **Timezone resolution:** корректное определение timezone по координатам и дате рождения — критически важно для точности расчёта. См. `docs/technical/architecture/timezone-resolution.md` для деталей.
