# Дизайн и UX

## Дизайн-философия

### «Космический минимализм»

Тёмная тема по умолчанию (cosmos feel), но с чистой типографикой и пространством. Не перегруженный, не «магический» — скорее как **Linear встречает Stellarium**.

| Принцип | Описание |
|---------|----------|
| **Dark by default** | Глубокий тёмный фон (#0A0A0F — почти чёрный с синим оттенком), не чистый #000 |
| **Типографика** | Чистый sans-serif (Inter/Geist) для UI, serif (Crimson Pro/Playfair) для эзотерических текстов |
| **Пространство** | Много воздуха. Контент дышит. Нет перегрузки элементами |
| **Цвет как акцент** | Основной UI — монохром. Цвет только для планет, знаков, акцентов |
| **Анимации** | Плавные, subtle. Framer Motion. Тайминг ниже в секции «Анимации и тайминг» |

### Progressive Disclosure

Сложность открывается постепенно:

- **Новичок:** видит простую карту, знак Солнца, Луны, восходящий + короткие описания
- **Продвинутый:** включает градусы, минуты, аспекты, дома
- **Эксперт:** полный расклад с орбисами, деканатами, фиксированными звёздами

Переключение через настройки + автоматический detection (если пользователь не взаимодействует с продвинутыми элементами — не показывать их).

### Mobile First

**80%+ пользователей** астро-приложений на мобильных. Дизайн начинается с 375px (iPhone SE), адаптируется вверх.

- Тач-жесты для колеса натальной карты (вращение, zoom)
- Навигация: нижний tab bar (5 вкладок максимум)
- Карточки контента вместо длинных страниц

### Навигационный принцип

**Колесо натальной карты — центральный hub.** Всё доступно через тапы по планетам/знакам на колесе:

```
       Тап на планету → эссе по позиции
       Тап на знак → описание знака
       Тап на дом → интерпретация дома
       Тап на аспект → описание аспекта
       Свайп → переключение sidereal/tropical
       Long press → детали (градусы, орбис)
```

### SEO-требования к типографике и контенту

- **H1** на каждой странице содержит primary keyword (например, «Sun in Sidereal Pisces»)

- **URL slugs:** чистые, keyword-rich — `/essays/sun-in-pisces-sidereal`, не `/essays/123` или `/essays?id=sun-pisces`

## Touch Targets

Все интерактивные элементы — **минимум 44×44px** (Apple HIG). Это критично для:

- Глифы планет на натальной карте (колесо) — даже если глиф 24px, tap area расширяется до 44×44px
- Кнопки в bottom tab bar
- Toggle sidereal/tropical
- Ссылки в тексте — line-height обеспечивает достаточную высоту, padding по бокам

## Анимации и тайминг

| Тип | Длительность | Easing |
|-----|-------------|--------|
| Micro-interactions (hover, press) | 150-300ms | `ease-out` |
| Page transitions | ≤ 400ms | `ease-out` для входа, `ease-in` для выхода |
| Exit animations | 60-70% от длительности входа | `ease-in` |
| Sidereal/tropical toggle | Spring physics | `spring({ stiffness: 300, damping: 25 })` |
| Chart wheel rotation | Spring physics | Естественное физическое ощущение |

**Правила:**

- Spring physics для элементов с «физическим» ощущением (вращение колеса, toggle, snap-back)
- Exit-анимации короче входных (60-70% длительности)
- `prefers-reduced-motion: reduce` — отключение всех non-essential анимаций. Оставляем только функциональные state changes (opacity toggle, instant position changes)
- Никогда не блокировать UI во время анимации

## Form UX

Форма ввода данных рождения — критичное первое взаимодействие. Ошибка здесь = потерянный пользователь.

| Правило | Описание |
|---------|----------|
| **Visible labels** | Всегда показывать label над полем, не только placeholder |
| **Inline validation** | Валидация на blur (потеря фокуса), не на каждый keystroke |
| **Сообщения об ошибках** | Под полем: причина + как исправить (например, «Date must be between 1800 and today») |
| **Семантические input types** | `type="date"` для даты, `type="time"` для времени, `type="text"` + autocomplete для города — мобильная клавиатура подстраивается |
| **Auto-focus при ошибке** | При submit с ошибкой — фокус на первое невалидное поле |

## Loading States

| Ситуация | Поведение |
|----------|----------|
| Расчёт карты (> 300ms) | Skeleton screen в форме колеса (не spinner) |
| Загрузка эссе | Skeleton для текста + заголовка |
| Multi-step операции | Progress indicator с текущим шагом |
| Короткие операции (< 300ms) | Ничего не показывать — мгновенный результат |

**Правило:** никогда не блокировать UI во время загрузки. Пользователь должен мочь вернуться назад или взаимодействовать с другими элементами.

## Empty States

| Состояние | Что показываем |
|-----------|---------------|
| Нет сохранённых карт | «Calculate your first natal chart» + CTA кнопка |
| Город не найден в базе | «City not found. Enter coordinates manually:» + поля lat/lon |
| Нет результатов поиска | Suggestions: проверьте написание, попробуйте другой запрос |

## Error UX: что видит пользователь когда что-то идёт не так

> Ошибки неизбежны. Хорошая обработка ошибок = доверие. Молчаливый провал = потерянный пользователь.

### Расчёт карты

| Ситуация | Что показываем | Действие |
|----------|---------------|----------|
| **Server timeout** (> 5s) | «Расчёт занимает больше времени, чем обычно...» + spinner | Автоматический retry × 1. Если 2-й раз timeout → «Попробуйте через минуту» |
| **Server 500** | «Не удалось рассчитать карту. Наша команда уже знает.» | Sentry alert. Кнопка «Попробовать снова» |
| **Offline** | «Для расчёта нужен интернет.» | MVP не включает offline-режим. При потере сети показываем стандартное браузерное поведение |
| **Невалидная дата** (будущая, < 1800) | Inline ошибка под полем: «Дата должна быть между 1800 и сегодня» | Zod validation на клиенте до отправки |

### Город / геокодинг

| Ситуация | Что показываем | Действие |
|----------|---------------|----------|
| **Город не найден** в 50K базе | «Не нашли [город]. Введите координаты вручную:» + поля lat/lon | Трекинг `city_search_failed` для расширения базы |
| **Полярная широта** (> 66.5°) | «Для полярных широт используем Whole Sign вместо Placidus» — info banner, не ошибка | Автоматический fallback на Whole Sign houses |

### Auth / Stripe

| Ситуация | Что показываем | Действие |
|----------|---------------|----------|
| **Clerk недоступен** | «Вход временно недоступен. Вы можете рассчитать карту без регистрации.» | Скрыть кнопку «Войти», показать «Рассчитать» |
| **Stripe checkout failed** | «Оплата не прошла. Попробуйте другую карту или вернитесь позже.» | Redirect back с error param. Не терять контекст |
| **Webhook delay** (premium не обновился) | «Оплата получена! Обновление статуса может занять пару минут.» | Polling `/api/user/status` каждые 10с × 6 раз |

### Общие принципы

1. **Никогда не показывать stack trace** или технические детали пользователю
2. **Всегда давать action** — кнопку «Попробовать снова», ссылку, или объяснение «когда это починится»
3. **Трекинг всех ошибок** через PostHog (`calculation_error`, `api_error`) + Sentry
4. **Graceful degradation** — если часть функциональности сломана, остальное работает (эссе из кэша, карта без домов)

## Pre-Delivery UI Checklist

Перед выпуском каждой страницы:

### Visual Quality
- [ ] Нет emoji вместо иконок (использовать Lucide React / кастомные SVG)
- [ ] Единая семья иконок (Lucide) и единый sizing
- [ ] Планетарные цвета соответствуют design tokens из этого документа
- [ ] Контраст тёмной темы: primary text >= 4.5:1, secondary >= 3:1

### Interaction
- [ ] Все touch targets >= 44x44px
- [ ] Press feedback на всех интерактивных элементах (opacity/scale)
- [ ] Тайминг анимаций 150-300ms, spring physics для естественного ощущения
- [ ] `prefers-reduced-motion` отключает non-essential анимации

### Forms
- [ ] Visible labels на всех inputs
- [ ] Inline validation на blur
- [ ] Сообщения об ошибках под полем с причиной + как исправить
- [ ] Auto-focus на первое невалидное поле при ошибке

### Layout
- [ ] Mobile-first проверено на 375px ширины
- [ ] Нет горизонтального скролла
- [ ] Bottom nav <= 4 items с icon + label
- [ ] Контент не скрыт за fixed bars

### Accessibility
- [ ] Последовательная иерархия заголовков (h1 -> h2 -> h3..., без пропусков)
- [ ] Skip to main content ссылка
- [ ] Клавиатурная навигация работает (Tab, Enter, Escape)
- [ ] Screen reader: у карты есть текстовая таблица-fallback
- [ ] Цвет — не единственный индикатор (подкреплён глифами/текстом)
