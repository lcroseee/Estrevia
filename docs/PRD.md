## 1. Executive Summary

> **Одно предложение:** Это платформа, где астрология встречается с реальной астрономией, а сообщество строится без алгоритмов и рекламы.

### Видение

Это первая платформа, которая объединяет профессиональную сидерическую астрологию, real-time астрономические данные и эзотерическую традицию (Thelema, Каббала, Tarot Thoth) с вирусным шерингом без алгоритмов и манипуляций.

**Миссия:** создать пространство, где люди изучают космос и себя, соединяясь друг с другом через настоящий интерес, а не через алгоритмическую ленту.

## 2. Взгляд Product Manager

### 2.1 Проблема

Существующие астрологические приложения страдают от трёх ключевых проблем:

- **Поверхностность:** Co-Star, The Pattern и другие используют только тропическую систему. Ни одно приложение не даёт сидерическую карту + эзотерический контекст (Древо Жизни, Thoth) в одном интерфейсе.
- **Разрыв между астрологией и астрономией:** Ни одно приложение не объединяет real-time научные данные (солнечные вспышки, тектоника, погода планет) с астрологической интерпретацией.
- **Токсичность соцсетей:** Люди устали от алгоритмов, рекламы и манипуляций. Растёт спрос на «чистые» платформы (BeReal, Mastodon, Bluesky).

### 2.2 Целевая аудитория

**Первичная (MVP)**

- Практикующие астрологи (сидерическая и тропическая традиция), 25-45 лет
- Изучающие Thelema, Каббалу, Tarot Thoth — ниша, но лояльная и платящая
- Любители астрономии, которых тянет к астрологии (и наоборот)

**Вторичная (Фаза 2-3)**

- Gen Z «астро-curious» — люди, которые знают свой знак, но хотят глубже
- Музыканты/художники, которым нужна платформа без алгоритма для продвижения
- Геймерские сообщества (замена Discord с более красивой кастомизацией)

### 2.3 Конкурентный ландшафт

| Продукт | Сидерич. | Viral Share | 3D космос | Эзотерика | Анти-алгоритм |
|---------|----------|------------|-----------|-----------|---------------|
| Co-Star | ✗ | Частично | ✗ | ✗ | ✗ |
| The Pattern | ✗ | Частично | ✗ | ✗ | ✗ |
| Moonly | ✗ | ✗ | ✗ | Частично | ✗ |
| Stellarium | ✗ | ✗ | ✓ | ✗ | N/A |

> **Ключевой инсайт:** ни один конкурент не закрывает более 2 из 5 категорий. Наш продукт закрывает все 5.

### 2.4 Позиционирование

> *«Это космическая вселенная, где настоящая наука встречается с древней мудростью, а люди находят друг друга через звёзды, а не через алгоритмы».*

> **КРИТИЧЕСКИЙ ВОПРОС PM:** Не пытаться построить всё сразу. Продукты, которые пытаются быть всем сразу, проигрывают всегда. MySpace начинал как простая страница + музыка. Spotify начинал как простой плеер.

## 3. Взгляд CTO: Техническая Архитектура

### 3.1 Общая схема

Приложение строится по модульной архитектуре с чётким разделением на домены. Каждый домен может быть разработан и развёрнут независимо:

| Домен | Ответственность | Сложность |
|-------|----------------|-----------|
| Astro Engine | Расчёты карт, эфемериды, сидер./тропич. | Средняя (Swiss Ephemeris) |
| Cosmos 3D | 3D-визуализация, планеты, созвездия | Высокая (Three.js / WebGL) |
| Data Feed | NASA, USGS, NOAA API интеграция | Низкая-средняя |
| Esoteric CMS | Crowley, Eshelman, Thoth, Tree of Life | Низкая (контент) |
| Social | Профили, лента, медиа, кастомизация | Очень высокая (Фаза 3) |
| Comms | Чат, звонки, группы | Очень высокая (Фаза 3) |
| Music | Уплоад, стриминг, плейлисты, лирика | Очень высокая + лицензии (Фаза 3) |
| AI Layer | Генерация картинок, профили, перевод | Средняя (API) |

> **Примечание:** Social, Comms и Music — это Фаза 3, не MVP. MVP фокусируется на Astro Engine, Esoteric CMS и Data Feed (Phase 2).

### 3.2 Критическая оценка CTO

> **РЕАЛЬНОСТЬ:** Строительство всего видения с нуля потребует 20-30 разработчиков и 3-5 лет. На MVP с 1-3 разработчиками нужно 4-6 месяцев. Это нормально. Все великие продукты начинали как минимальная версия.

**Принципы архитектурных решений:**

- **Build vs. Buy:** Чат/звонки — не писать с нуля. Использовать Stream, Sendbird или Matrix. Музыкальный стриминг — только сторонние CDN.
- **Modular First:** Каждый домен — отдельный микросервис. На старте монолит, но с чёткими границами модулей.
- **API First:** Всё через REST/GraphQL API. Это позволит позже легко добавить мобильное приложение, виджеты, внешние интеграции.

### 3.3 Верификация астрологических расчётов

> **Точность расчётов — критический фактор доверия. Одна ошибка в градусах = потеря целевой аудитории практикующих астрологов.**

**Стратегия верификации:**

1. **Эталонное сравнение:** каждый расчёт Estrevia сверяется с Astro.com (бесплатный API), Solar Fire и Jagannatha Hora. Допуск: ±0.01° для планетарных позиций.
2. **Автоматические тесты:** CI/CD pipeline включает тест-сьют из 100+ эталонных карт (известные даты рождения с верифицированными позициями). При каждом деплое автоматически проверяется точность.
3. **Ayanamsa валидация:** сидерическая карта зависит от выбранной ayanamsa (Lahiri, Fagan-Bradley, Krishnamurti). Estrevia поддерживает все три, с Lahiri по умолчанию. Каждая верифицируется отдельно.
4. **Beta-тестирование:** 50+ практикующих астрологов проверяют свои карты перед публичным запуском. Форма обратной связи для сообщения о расхождениях.
5. **Отображение источника:** в интерфейсе указывать 'Swiss Ephemeris v2.10 | Ayanamsa: Lahiri' — это стандарт прозрачности в астрологическом сообществе.

### 3.4 PWA

MVP: PWA manifest для installability (иконка на домашнем экране, standalone окно). Никакого офлайн-кэширования — все функции требуют сети. Фаза 2: оценить Service Worker кэширование для эссе.

### 3.5 LLM-генерация контента (Claude API)

> **120+ астрологических текстов (10 планет × 12 знаков) генерируются через Claude API с экспертным промптом, затем верифицируются практикующим астрологом.**

**Пайплайн генерации контента:**

1. **Системный промпт:** включает контекст из public domain источников (777, Equinox), определения сидерических позиций, стилистические указания (глубина Eshelman, поэтика Crowley).
2. **Генерация:** Claude API (claude-sonnet-4-20250514) генерирует 120 текстов по 300-500 слов каждый. Стоимость: ~$1.50 (120 текстов × $0.012/текст). English only.
3. **Качество:** каждый текст проходит через второй LLM-вызов для fact-check (соответствие астрологическим традициям). Несоответствия помечаются для ручной проверки.
4. **Обновление:** тексты хранятся как MDX файлы в репозитории. Фаза 2: headless CMS.

## 6. MVP Скоуп и Приоритизация

> **MVP = то, что делает одну вещь лучше всех. Для нас это: лучшая сидерическая натальная карта с эзотерическим контекстом.**

### Что входит в MVP (8-12 недель)

1. Натальная карта с переключателем сидерическая / тропическая
2. Real-time планетарные часы (астрологическое колесо в реальном времени)
3. Лунный календарь: фаза, знак, положение
4. Эссе Eshelman и Crowley по каждой планетарной позиции
5. Объяснение почему сидерическая система точнее
6. Регистрация и сохранение карты
7. «Космический паспорт» — viral share card + OG image
8. English only (без i18n для MVP)

### Что НЕ входит в MVP (и почему)

- **Социальная сеть:** Сначала нужны пользователи. Соцсеть без людей — мёртвый продукт. Сначала привлекаем аудиторию астрологией, потом добавляем соц.слой.
- **Музыка/стриминг:** Это отдельный бизнес. Лицензионные соглашения стоят миллионы. На старте — интеграция Spotify/YouTube вместо собственного стриминга.
- **Чат/звонки:** Требует отдельной команды. На старте — ссылки на Telegram/Discord сообщества.
- **3D-вселенная:** Красиво, но не решает проблему пользователя. Добавляем после product-market fit.
- **Онлайн-библиотека:** Юридический кошмар. Начинаем с public domain текстов (Crowley, Eshelman, Liber 777 — многое уже в public domain).
- **i18n:** English only для MVP. Испанский и другие языки в Фазе 2.
