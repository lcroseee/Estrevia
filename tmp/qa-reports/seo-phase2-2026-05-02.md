**~~🔴 HIGH BUG~~ ✅ FIXED — SunSignWidget API response mismatch:**

File: `src/app/[locale]/(app)/sidereal-[sign]-dates/SunSignWidget.tsx`

**Bug (original):** Line 59 called `setResult(await res.json() as SunSignResult)`. TypeScript `as` cast is compile-time only — does not reshape runtime object. API returns `ApiResponse<SunSignResult>` wrapper `{ success, data: { sign, ... }, error }` but widget read `result.sign` directly off the wrapper → `undefined`.

**Impact (before fix):**
1. `result.sign` → `undefined` → `t('signs.undefined')` → missing translation key
2. `new Date(undefined)` → Invalid Date → garbled date display
3. `result.sign !== currentSign` → `undefined !== 'aries'` = `true` → cross-link ALWAYS renders
4. Cross-link `href="/sidereal-undefined-dates"` → 404

**Fix (commit `31564a7`):** Unwrap `body.data`; add null guard; import `ApiResponse<T>` from `@/shared/types/api`.

```ts
const body = await res.json() as ApiResponse<SunSignResult>;
if (body.success && body.data) {
  setResult(body.data);
} else {
  setError(t('widgetError'));
}
```

TypeScript strict: 0 errors post-fix. Bug is CLOSED.
